# Zig 0.16 verification for `blitz`

Compact verdicts. ✓ = confirmed. ✗ = corrected.

- ✓ 0.16.0 stable release date: 2026-04-13. Source: https://ziglang.org/download/
- ✓ main entry: `pub fn main(init: std.process.Init.Minimal) !void` still valid in 0.16. Source: https://ziglang.org/download/0.16.0/release-notes.html | https://ziglang.org/documentation/master/
- ✓ latest nightly naming: `std.Io.Threaded` still named `Threaded`; `std.Io.Evented` still exists too. Source: https://raw.githubusercontent.com/ziglang/zig/master/lib/std/Io.zig
- ✓ 0.16 I/O split: `Io.Threaded` = feature-complete / well-tested; `Io.Evented` = work-in-progress / experimental. Source: https://ziglang.org/download/0.16.0/release-notes.html
- ✗ blocking `std.fs` / `std.process` as primary new-code path: corrected to `std.process.Init` + `std.Io` APIs; 0.16 release notes map many `std.fs` ops to `std.Io.*`, and `Io.Threaded` is the 0.15.x-compatible path. Source: https://ziglang.org/download/0.16.0/release-notes.html
- ✗ build.zig API: `addExecutable`, `addLibrary`, `addModule` live on `std.Build`; `addCSourceFile`, `linkLibC`, `addIncludePath` are deprecated on `Compile` and should move to `root_module.*`. Source: https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/Build.zig | https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/Build/Step/Compile.zig
- ✗ cross-compile claim: corrected to `zig build -Dtarget=<target-query>` via `standardTargetOptions`; verified official examples/target table show `aarch64-macos`, `x86_64-macos`, `x86_64-linux`, `aarch64-linux`, `x86_64-windows`. Musl ABI target strings were not explicitly captured in loaded 0.16 excerpts. Source: https://ziglang.org/learn/build-system/ | https://ziglang.org/download/0.16.0/release-notes.html
- ✗ `std.fs.cwd().atomicFile(path, .{})`: corrected to pass `AtomicFileOptions.write_buffer` (required), e.g. `std.fs.cwd().atomicFile(path, .{ .write_buffer = &buf })`. Source: https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/fs/Dir.zig | https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/fs/AtomicFile.zig
- ✗ allocator pattern: corrected; `GeneralPurposeAllocator` is deprecated alias to `DebugAllocator`. For short-lived CLI, use `std.process.Init`’s `gpa`/`arena`, or explicit `std.heap.DebugAllocator(.{}){}` + `std.heap.ArenaAllocator`. Source: https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/heap.zig | https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/heap/debug_allocator.zig
- ✓ `.zig-version` pin: conventional in Zig version managers; zvm resolves from `ZIG_VERSION`, `.zig-version`, or `build.zig.zon`. Source: https://raw.githubusercontent.com/Turrain/zvm/4ea4ceb237a973760084e7b67a8ed9ea6271f1a9/README.md | https://raw.githubusercontent.com/Turrain/zvm/4ea4ceb237a973760084e7b67a8ed9ea6271f1a9/src/resolve.zig
- ✓ `zig build --watch -fincremental`: official 0.16 release notes say `zig build -fincremental --watch` is usable. Source: https://ziglang.org/download/0.16.0/release-notes.html
- ✓ `std.process.Child` exists in 0.16. Source: https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/process/Child.zig
- ✓ `std.json` module exists in 0.16. Source: https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/json.zig
- ✓ `std.crypto.hash.sha2.Sha256` exists in 0.16. Source: https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/crypto.zig | https://raw.githubusercontent.com/ziglang/zig/738d2be9d6b6ef3ff3559130c05159ef53336224/lib/std/crypto/sha2.zig
- ✓ real 0.16 tree-sitter Zig example found: `tree-sitter/zig-tree-sitter` says “Needs zig 0.16” and uses `standardTargetOptions` + dependency/link flow. Source: https://raw.githubusercontent.com/tree-sitter/zig-tree-sitter/master/README.md
- ✓ hermetic tree-sitter grammar compile pattern found: real Zig build files vendor `src/parser.c` via `root_module.addCSourceFile` + `addIncludePath` (no system tree-sitter required in those examples). Source: https://raw.githubusercontent.com/ziex-dev/ziex/26f8b93475c0c77b6455565c5368e3b252b62b4f/pkg/tree-sitter-zx/build.zig | https://raw.githubusercontent.com/tree-sitter/tree-sitter/457eb295b758a579059d8e9a6c7d5f0fceca1f0a/build.zig
- ✗ `@cImport` in 0.16: future direction says deprecated; prefer build-system C integration for C headers. This matters if `blitz` tries to lean on `@cImport("tree_sitter/api.h")`. Source: https://ziglang.org/download/0.16.0/release-notes.html
